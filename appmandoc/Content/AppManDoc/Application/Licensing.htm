Licensing

Many commercial plugins for OpendTect use FlexLM licensing. The FlexLM end-user manual can be found at http://opendtect.org/lic/doc/flexnet_end_user_manual.pdf.

 

The main source of information from opendtect.org is http://opendtect.org/lic/doc/flexnet_installation_guide.html.

 

Some hints: There are two types of licenses: 'node-locked' and 'floating':

 

A node-locked license is locked to a certain machine. Node-locked licenses have only got lines like this:

 

# Dip-Steering

FEATURE dTectDS dgbld 5.000 15-jan-2015 uncounted B4909ACBA201 \

HOSTID=81537ee56k1g SIGN="003D 554A E7AA 524C 4143 AJ7H DA91 2C00 A967 \

214D B70A 6983 9435 78A9 D947"

 

# Neural Networks

FEATURE dTectNN dgbld 5.000 15-jan-2015 uncounted 3DA0D1CBA2D2 \

HOSTID=81537ee56k1g SIGN="00BC AC49 AJ7H E557 BA20 7444 57D5 4C00 831E \

4072 F677 0831 1FB9 4D38 BB7E"

 

Also demo licenses (these have HOSTID=DEMO but look the same) are handled in this same way.

 

 

Another kind of license file ('floating license') require a machine designated as license server. On this machine, a daemon (UNIX) or service (Windows) will keep track of the licenses available, therefore the DAEMON line will need to be update to the full path to the location of the license daemon including the daemon itself and its extension. These are typically used when OpendTect can run on multiple computers. These license files look like this:

 

SERVER myserver b9mk6f97036p

DAEMON dgbld full_path_to/dgbld

 

FEATURE dTect dgbld 5.000 25-jun-2015 8 AE0FBB71CE46 DUP_GROUP=D \

SIGN="0098 0DD9 3ED8 7BA4 JJ57 F8DC 69A1 0A00 8145 F528 6A2E \

2237 0071 8AC0 469D"

 

# Dip-Steering

FEATURE dTectDS dgbld 5.000 25-jun-2015 8 07408D0E40F9 DUP_GROUP=D \

SIGN="0003 CCD5 JJ57 07CB C551 1877 AFD0 8F00 79AB A687 9843 \

B83C 9405 67F8 B992"

 

The file is read like this: FlexLM needs to start a server on machine 'myserver' with FlexLM host ID b9mk6f97036p. To start this server, use a program lm_dgb(.exe). Until 25-Jun-2015 you can use OpendTect and the plugin 'Dip-Steering' (dTectDS) with up to a maximum of 8 concurrent users, on version 5.0.x (and this is also back-compatible to OpendTect 4.6.0x).
