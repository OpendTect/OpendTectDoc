4.2 Setting up the environment

OpendTect source code

To build OpendTect from source you need to pull the source code from https://github.com/OpendTect/OpendTect. Then build the solution with CMake. In this case launch CMake and browse to your OpendTect source folder and start configuring and generating from CMake. CMake will prompt for Qt and OpenSceneGraph directory location, select them from CMake itself and continue. Once the projects and solution are generated you can start building OpendTect.

You can use the following git commands to clone the repositories to your local machine.

git clone https://github.com/OpendTect/OpendTect.git --single-branch --branch main od

git clone https://github.com/OpendTect/OpendTect.git --single-branch --branch od6.6 od6.6

CMake

Download CMake from https://cmake.org/download/.

Qt

Go to https://www.qt.io/download to download the required QT binaries by using their online installation tool. Be sure to select only the components you require, as it is very space consuming to install all components of QT.

For building the main, od6.6.0 and od6.6 branch QT 5.15.0 is required. For branches od6.4.5, od6.4 and od6.5 QT 5.9.6 is required.

For building with Visual studio select the MSVC 2019 64-bit component (Windows), SDK 10.14 (MacOS) or gcc 64 bit (Linux). Also select the appropriate Q WebEngine for installation in the same version. Optionally you can also install source code and/or debug information files.

OSG

Get the OSG binaries here: https://objexx.com/OpenSceneGraph.html. The 3.6.5 release and/or debug version is needed for building the main, od6.6.0 and od6.6 branches of OpendTect. The 3.6.3 release and/or debug version is needed for building the od6.4.5, od6.4 and od6.5 branches.

HDF5 (optional)

Get HDF5 here: https://www.hdfgroup.org/downloads/hdf5. The link to HDF5 requires to provide the path to an existing HDF5 installation. All versions above 1.10.3 are supported, but using the current API 1.12 is preferred. Installation is best done using their binary installations (on Windows especially), or from the package manager on Linux. Windows developers however need to recompile the sources since no debug binary libraries can be downloaded.
