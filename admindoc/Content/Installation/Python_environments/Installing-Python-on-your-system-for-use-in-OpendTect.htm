Installing Python on your system for use in OpendTect

It is possible to use a system Python installation in OpendTect. In this scope such a Python installation is not a miniconda or anaconda installation, and is generally installed by the System Administrator or by the user.

In order for OpendTect to use a system Python installation the following things need to be done:

  • Several packages need to be installed with pip, we will give a list below

  • Non Python libraries CUDA, cuDNN and HDF5 need to be installed

  • Make sure that PATH is set correctly for Python and the non Python libraries

Python supported version

Currently OpendTect and the Machine Learning plugin supports Python 3.8.

Windows recommendations

  • To check whether Python has been installed open the Command Prompt and run:
    where.exe python
    If it has been installed it will give the path to python.exe

  • We recommend to install Python with the full installer from Python.org Downloads page for Windows. The other options to install Python are from the Microsoft Store and via the Windows Subsystem for Linux (WSL). The reason for not recommending the Microsoft Store package is that it has limitations that make it not suited for a professional development environment.

  • Make sure that you toggle off App Installer python.exe and App Installer python3.exe in the App execution aliases settings.


    Figure: Settings > Manage app execution aliases

Installations with pip

In the Command Prompt or Terminal run:

pip install --upgrade pip
pip install wheel
pip install python-docx python-pptx
pip install safety
pip install bokeh==2.4.2 h5py humanfriendly jupyterlab keras matplotlib numba numpy onnx onnxmltools onnxruntime pandas psutil scikit-learn scipy spyder tensorboard==2.7.0 tensorflow==2.7.0 torch torchaudio torchvision xgboost

Installing CUDA Toolkit, cuDNN and HDF5

CUDA Toolkit can be downloaded from the CUDA Toolkit download page. Currently we support CUDA Toolkit 11.3.

cuDNN can be downloaded from the cuDNN Archive. Currently we advise to download and install cuDNN v8.2.1 (June 7th, 2021), for CUDA 11.x

HDF5 can be downloaded from the hdfgroup.org downloads page. Currently we advise to download and install HDF5 1.12.2.

Check whether all paths are set

The paths to the Python and Python Scripts directory should have been set in environment variable Path, e.g:

  • C:\Users\your_username\AppData\Local\Programs\Python\Python38

  • C:\Users\your_username\AppData\Local\Programs\Python\Python38\Scripts

The following paths should be listed in the System Environment Path variable for Windows:

  • C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.3\bin

  • C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.3\extras\CUPTI\lib64
    Please note that this path the CUDA Toolkit installer will not set. It is recommended to set this one manually so that Python can use CUPTI.

  • C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.3\libnvvp

  • C:\Program Files\HDF_Group\HDF5\1.12.2\bin

Use the system Python environment in OpendTect

To use the system Python environment select System as Python environment in the Set Python settings window. When the system Python has been succesfully installed the Test button should give the list of Python packages that are available in the system Python environment.

If the Test button gives the below error we suggest that you first check whether App Installer python.exe and App Installer python3.exe in the App execution aliases settings are toggled on. If so then toggle them off and run the test again.
