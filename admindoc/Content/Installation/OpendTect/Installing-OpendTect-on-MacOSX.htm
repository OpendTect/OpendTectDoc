Installing OpendTect on Mac OS X

OpendTect, the dGB Commercial plugins, Python packages and offline User documentation and dGB Plugins documentation can be installed via the OpendTect Installation Manager (OpendTect_Installer_mac.dmg) or via an offline package.

There are some improvements in the installation manager:

  • Installation packages are now grouped, e.g. all Python related packages are listed in group Python

Security setting

Please do the following before installation:

  • Click on the 'Apple' button (top left), go to:
  • System Preferences > Security & Privacy > General Tab
  • In case the option is there then toggle on Allow apps to be installed from anywhere, this may ask for admin credentials.
  • The option Allow apps to be installed from anywhere will not be there in recent Mac OS X versions. In that case the first time you run the OpendTect installer it will not be allowed to run. first time you run the OpendTect installer there will appear an option to trust the OpendTect installer when running the installer. Please do so when this option appears.
  • Then run the installer. If the installer is not allowed to run then go into the above mentioned Security & Privacy > General Tab and look for a mention about the OpendTect Installer. If it is there then choose to allow it to run.
  • Once the installation is finished, and assuming you made a change to the security setting then you may change the security back.

Installation via Installation Manager

  1. The Installation Manager is available for download via the appropriate platform link on the download page of the dGB website.

  2. Run the .dmg file via Finder.

  3. Then start the OpendTect Installer.

    Figure: Start the OpendTect Installer by opening the .dmg file

  4. The Installation Manager is a wizard to install/update the existing OpendTect (Current / Preview) releases. The release type field is used to select the release that is needed to be installed/updated. The installer gives you the choices as seen below:

    Figure: OpendTect Installation Manager

  5. If it is a fresh installation then you may keep the proposed installation base directory or choose your own. If it is an update then select it's installation base directory. Now click Proceed.

  6. Select the package type of OpendTect you want to install. To read more about OpendTect packages types, please refer to our webpage of licensing types.

    Figure: OpendTect Installer Package Selection Window

  7. The OpendTect Installation Manager identifies the platform on which it is running. This information is then anonymized prior to it being sent to OpendTect. We use this anonymous data solely for the purpose of getting a picture of OpendTect usage and thus improving our support capabilities.|

  8. The next window of the wizard is the OpendTect Package Manager. Multiple items can be toggled on or off by checking the boxes. Optionally, the relevant package combination could also be selected from the top list box.

    Figure: OpendTect Package Manager

  9. The installation manager will automatically recognize the previously installed version at the selected path and will prompt it in the Installed version field. To read more about a particular item in the list, select the item by clicking on it and read the description on to the right panel. For example, Dip-Steering:

  10. After selecting the packages for installation click on Proceed to start the installation. You will first get a window that downloads the files from the download server. After downloading the packages they will be unpacked to the installation path.

  11. If, for any reason, you should choose to abort the installation mid-download, you will see the following window appear:

    This gives you various options, including increasing the time-out from its default setting, changing the download server or changing the Proxy settings.

  12. The installation will finish with giving an overview of the installed packages.
