Environment variables

There are several environment variables that can be set. OpendTect will look for these environment variables. They can be set to get special features working or to use custom paths for certain functionality.

Environment variable Expected Values Description
DTECT_BATCH_HOSTS_FILEPATH Path to a BatchHosts file This sets the path to a tailor-made BatchHosts file. If it is not set then OpendTect will use the default BatchHosts file in the OpendTect installation directory, e.g. C:\Program Files\OpendTect\6.6.0\data\BatchPrograms .
DTECT_CLUSTER_PROC yes or no When set to yes OpendTect will have cluster processing enabled.
DTECT_SETTINGS Path to the OpendTect settings home directory This sets the path to the OpendTect settings home directory. Use this if you do not want to use the .od directory in the user's home folder. Use this environment variable for Linux and Mac.
DTECT_WINSETTINGS Path to the OpendTect settings home directory This sets the path to the OpendTect settings home directory. Use this if you do not want to use the .od directory in the user's home folder. Use this environment variable for Windows.
LM_LICENSE_FILE    
OD_USE_VIRTUALMEM yes or no When set to yes OpendTect will support using virtual memory.
RSFROOT Path to the installation directory of Madagascar This sets the path to the Madagascar installation directory. If it is not set then and one would try to start Madagascar from OpendTect an error message would be displayed.
