4.4.8.1 Export SEG-Y

2D/3D Poststack and 3D Prestack data can be exported from OpendTect in SEG-Y format:

The SEG-Y revision 1 default bytes locations will be used during export, but additional positions can be used with the personal setting keywordspersonal setting keywords listed on the right-hand side.

The pick is a trace attribute stored in the OpendTect seismic files. It is most often not used.

The reference number is most often the Shot Point for 2D data but could be used for anything else. Please note that a SEG-Y scalar at bytes 201-202 apply for values stored in bytes 197-200. The SP scalar will always be -10, thus the value written on bytes 197-200 is 10 times the SP value.

All values listed above are encoded on 4 bytes by default. The byte length can be overridden using the following personal key words, except for the coordinates:

  • SEG-Y.Nr bytes for In-line
  • SEG-Y.Nr bytes for Cross-line
  • SEG-Y.Nr bytes for Offset
  • SEG-Y.Nr bytes for Azimuth
  • SEG-Y.Nr bytes for trace number
  • SEG-Y.Nr bytes for Pick
  • SEG-Y.Nr bytes for RefNr

The layout of the SEG-Y export window changes slightly based on the data type.
