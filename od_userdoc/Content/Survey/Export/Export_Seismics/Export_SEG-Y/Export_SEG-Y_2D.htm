4.4.8.1.2 Export SEG-Y 2D

Stored 2D data can be exported from OpendTect in SEG-Y format.

All fields are optional, except the output filename that must be provided.

  • Input Data Set: Select the data set to be exported.
  • Line name: The lines available in the data set are listed here. Only one line can be selected.
  • Trace subselection: Can be defined here in various ways.
  • Null traces: They can be either discarded or written in the SEG-Y file.
  • Scale values: A linear scaling can be applied while exporting the data: a simple linear scaling that’s applied to the trace values. Formula is: y=ax+b where a is the factor, b is the shift, x is the original value, y is the new scaled value. Useful when storing 32 bit data as 16 bit.
  • Export to other CRS: By default the SEG-Y will be exported in the CRS of the current survey. Choose 'No' to select a different coordinate system for the output file. (The option is available only if the current survey has a defined projection based coordinate system).
  • SEG-Y format: Please note that this option may clip your data if the output format has less bytes than the input OpendTect format.
  • Text header: The SEG-Y textual header is automatically created, but may be provided by the user, either from a text file or directly from a SEG-Y file.
  • Export more lines from the same dataset: Checking this option removes the Line Name and Trace Subselection options from the main window (above) and, on pressing Export, the 2D SEG-Y multi-export window appears wherein the specify the file details to be exported:

Pressing OK will then, in one batch, export each selected line from the dataset as an individual SEG-Y file to the specified location.
