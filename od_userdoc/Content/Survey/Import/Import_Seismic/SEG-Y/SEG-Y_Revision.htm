4.3.10.1.2 SEG-Y Revision

The table in the Import SEG-Y Data window is auto-filled for a user by a quick scan of SEG-Y byte and trace headers. Actually use column lists suggested parameters for reading a SEG-Y file and Quick scan result displays scanned values for QC. Inspect these columns, do any appropriate changes in Actually use column and QC them in Quick scan result.

SEG-Y Revision (default=value in byte 301 of the SEG-Y binary header):

  • SEG-Y Rev. 0:
    • "traditional" SEG-Y conforming to the 1975 standard.
    • Data format, Number of samples, Z Range start/interval can be overruled.
    • Data positioning: bytes for applicable positioning info (IL/XL, Trace/SP, X/Y and offset) must be selected by a user.
  • SEG-Y Rev. 1:
    • Data format, Number of samples, Z Range start/interval can be overruled.
    • Data positioning: bytes for applicable positioning info (IL/XL, Trace/SP, X/Y and offset) are read from standard locations as described in the SEG standardsSEG standards
  • SEG-Y Rev. 2:
    • Data format, Number of samples, Z Range start/interval can be overruled.
    • Data positioning: bytes for IL/XL, Trace/SP, X/Y and offset are read from standard locations as described in the SEG standardsSEG standards

Data format (default=read from byte 25 of binary header): (optionally) overrule data format.

Most header values and data samples are written using several bytes for each word/sample. Therefore a correct byte order is a necessity. All SEG-Y standards (Rev. 0, 1 and 2) require using big-endian byte ordering. However, some vendors would still violate this requirement and use little-endian (reverse) byte order. Using standard SEG-Y data formats for reading such data results in unexpected scanned values of trace headers and unexpectedly large sample values (check the histogram). In this case use data formats with (byte swapped) option.

Number of samples (default=read from byte 21 of binary header): (optionally) overrule the number of samples per trace.

Z Range: in seconds or meters. Negative times are allowed.

  • start (default=read from trace headers laga and delrt): (optionally) overrule start of Z-range (in seconds or meters, negative start is allowed).
  • interval (default=read from byte 17 of binary header): (optionally) overrule Z-sampling, i.e. sampling rate (in seconds or meters)

Data positioning:

3D post-stack data:

  • In-line range (default=read from trace header byte 189)
  • Cross-line range (default=read from trace header byte 193)

or

  • X-Coordinate range (default=read from trace header byte 181)
  • Y-Coordinate range (default=read from trace header byte 185)

2D post-stack data:

  • Trace number range (default=read from trace header byte 21):
    • In file: from a specified trace header byte.
    • Generate: generate trace numbers by providing the number of the first trace in a 2D line and step in trace numbers.
  • Shot-Point number range (default=read from trace header byte 197)
  • X-Coordinate range (default=read from trace header byte 181)
  • Y-Coordinate range (default=read from trace header byte 185)

2D and 3D pre-stack data:

  • Offset range (default=read from trace header byte 37)
    • In file: from a specified trace header byte.
    • From Src/Rcv (X/Y): calculate from source and receiver X/Y coordinates (standard byte locations 73 and 77 for source, 81 and 85 for receiver are used).
    • Generate: generate offsets by providing the offset of the first trace and step in offset values.
