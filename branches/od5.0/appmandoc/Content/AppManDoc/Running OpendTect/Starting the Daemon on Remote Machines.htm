Starting the Daemon on Remote Machines

 

On Windows:

Launch the daemon on every remote machine by selecting

 

C:\ProgramData\Microsoft\Windows\Start Menu\Programs\OpendTect\5.0.0\Remote Processing Service

or

C:\Program Files\OpendTect\5.0.0\bin\win64\rsm\od_remote_service_manager.exe

 

This tool will add a notification icon to the system tray and pop up this window:

 

 

Press 'Start'... the remote machines are now ready to be used as multi-machine processing nodes. Once launched, the service can also be controlled by right-clicking on the system tray icon and selecting appropriate menu item like start, stop or exit:

 

 

 

 

On Linux:

Start the daemon by using:

$DTECT_APPL/exec_prog od_remoteservice

or via

Utilities > Tools > Batch programs... select [Another OpendTect batch program] from the drop-down list and, in the 'Program name' field, enter od_remoteservice and press Run. This process can be stopped via the 'Progress' window that pops up.

 

 

 

The daemons (on both Windows and Linux) should only be started on the computing nodes after:

  • configuring the BatchHost file on the server
  • setting its location with the environment variable (if not using the default location at C:\Program Files\OpendTect\5.0.0\data\BatchHosts_5.0).
  • The DTECT_DATA location, stored in the user settings, should match the value written in the BatchHosts configuration file. The easiest way for doing that is to start the OpendTect application and set the "Survey Data Root" to the correct path. OpendTect may be closed thereafter. Also one can change the Survey Data Root path on the computer nodes after the daemon has been started, without corrupting the process.

 

Any change in these three conditions requires only Stop and re-Start of the remote service.
